## 🚀 如何使用

### 启动应用

```bash
npm run dev
```

### 登录系统

1. 打开浏览器访问：http://localhost:5173/
2. 使用上面任意一个测试账号登录
3. 开始测试各项功能

### 测试功能

#### 1. 概览页
- 查看项目统计数据
- 查看项目类型分布
- 查看项目状态趋势

#### 2. 项目列表
- 查看所有项目
- 搜索项目（按名称）
- 筛选项目（类型、状态、优先级）
- 点击项目名称查看详情

#### 3. 创建项目
- 点击「创建项目」按钮
- 填写项目信息
- 提交保存（会立即显示在列表中）

#### 4. 编辑项目
- 在项目详情页点击「编辑」
- 修改项目信息
- 保存更改

#### 5. 删除项目
- 在项目详情页点击「删除」
- 确认删除（会从列表中移除）

#### 6. 主题切换
- 进入设置页
- 选择主题：超极简/玻璃拟态/暗黑
- 立即生效

#### 7. 语言切换
- 进入设置页
- 选择语言：中文/English
- 立即生效

## 🔧 Mock 模式配置

Mock 模式通过环境变量控制：

### 启用 Mock 模式（默认）

`.env.development` 文件：
```env
VITE_USE_MOCK=true
```

### 禁用 Mock 模式（连接真实后端）

修改 `.env.development` 文件：
```env
VITE_USE_MOCK=false
VITE_API_BASE_URL=http://your-backend-api:3000/api
```

重启开发服务器后生效。

## ⚠️ 注意事项

1. **数据不持久**：刷新页面后，创建/修改的数据会重置为初始状态
2. **文件上传**：Mock 模式下文件上传功能暂不可用
3. **附件功能**：项目附件仅显示示例数据，无法实际上传
4. **用户搜索**：创建项目时的人员选择功能需要后端支持

## 🔄 切换到真实后端

当后端 API 准备好后：

1. 修改 `.env.development`：
   ```env
   VITE_USE_MOCK=false
   VITE_API_BASE_URL=http://your-backend:3000/api
   ```

2. 重启开发服务器：
   ```bash
   # 停止当前服务器（Ctrl+C）
   npm run dev
   ```

3. 系统会自动连接真实 API
